記録の追加・編集

下のタブで記録の追加・編集とエリア・植栽名のマスタ編集を切り替えられます。一覧の閲覧は閲覧ページから。

保存先への接続

新しい記録を追加

この記録を編集しています。保存で上書きするか、「編集をやめる」で取り消せます。写真を選ばなければ今の画像のままです。

植栽（複数選択可）
写真（任意）

スマホでは次のどちらかから選べます（同時に選べるのは1枚だけです）。

写真は長辺おおよそ 1024px に縮小して JPEG 保存します。角庭のように一覧に無い植栽は「その他」に入力してください。

バックアップ

記録を JSON でダウンロードできます（閲覧ページと同じ内容です）。

エリア

植栽名（マスタ）

保存には上の「アップロード用トークン」と同じ合言葉が必要です。各エリアのエリアIDは英小文字・数字・ハイフンのみ（例: north-garden）です。IDを変えると、既存の成長記録のエリア紐づけも保存時に更新されます。

保存には上のトークンが必要です。チェックボックス用の名前を行単位で編集します。名前を変えた行について、同じエリアの成長記録に含まれる旧名を新名へ置き換えます（行の追加・削除や並べ替えだけでは記録は自動では変わりません）。

記録にだけ古い名前が残っているとき（任意）

一覧の名前はすでに新しくても、過去の記録だけ旧表記のまま、というときに使います。「マスタをサーバーに保存」と同時に、指定したエリアの記録だけ置き換えます。

エリア・植栽名マスタ

画面上部のエリアタブと植栽名タブで、それぞれエリアの追加・削除・ID と、チェックボックス用の植栽名の行を編集します。保存内容は植栽一覧ページと成長記録フォームの両方に反映されます。データはサーバー（KV）に保存され、リポジトリの data/plants.json より優先されます。

エリアは「エリアを追加」で増やしたり、表示名・エリアIDの編集、「このエリアを削除」で減らせます（エリアは最低1つ必要です）。エリアIDを変えて保存すると、紐づく成長記録のエリアIDも更新されます。

既存の成長記録に載っている植栽名を変えたときは、同じエリア（変更後のID）の記録だけ、旧名を新名に置き換えます。行の追加・削除や並べ替えだけでは、記録側の名前は自動では変わりません。

このページについて

入力した写真やメモはサーバーに保存されます。閲覧だけならトークン不要のトップページ（index.html）を使えます。

記録の閲覧と写真の表示はどの端末からでも行えます。このページでの保存・編集・削除・マスタの保存だけ、トークンが設定されている場合は上の欄への入力が必要です。作業はタブ（記録／エリア／植栽名）で切り替えられます。

ブラウザのアドレスが http や https で始まる形で開いている必要があります。

保存先への接続

この欄の下に表示されるメッセージが、サーバーとのつながり具合を示します。

写真は公開URLまたは画像用API経由で配信されるため、閲覧だけならトークンは不要です。トークンはこのページの書き込み用です。

アップロード用トークンとは

サイトの管理者がサーバー側で設定した「合い言葉」のようなものです。設定されている場合は、ここに同じ文字列を入力して「トークンを保存」してください。知らない人が勝手に記録を書き換えるのを防げます。

入力した内容はこのブラウザにだけ保存され、サーバーには自動では送られません（保存や表示のときに必要な通信で使われます）。

管理者がトークンを設定していない場合は、この欄は空のままでかまいません。

リンク集のURLを設定してください

「リンク集へ戻る」は、入口のトップページへのリンクです。インターネット上の本番サイトでは、次のいずれかで先のアドレスを指定してください。

  • data/hub-link.json の linkCollectionUrl に、リンク集のページのURL（https://… で始まるもの）を書く
  • または、HTML の先頭付近の window.__PLANTING_HUB_URL__ に同じURLを書く

自分のパソコン上のフォルダだけで見ているときは、フォルダの並びに合わせて自動で開くようになっています。
